# CopiMine: финальная ручная проверка админом

Этот файл нужен для первого полного теста после переноса на Ubuntu. Колонку **Фактический результат** специально оставляй пустой до проверки: туда админ вписывает, что реально произошло.

## Что добавлено и как работает

- `CopiMineUltimateAdminPlus` управляет банком, AR, PIN, PostgreSQL, выборами, sidebar, админскими GUI и официальным bridge.
- `CopiMineArtifacts` работает отдельным плагином лавки артефактов. Он не меняет банк напрямую: покупка, возврат и repair идут только через bridge основного плагина.
- `CopiMineNarcotics` добавляет игровой RP-чёрный рынок с вымышленными предметами. Крафтов и админ-выдачи нет, покупка идёт только через AR bank bridge.
- Банк и лавка разделены: банк пополняет/снимает/переводит AR, лавка только показывает товары и покупает через банк.
- PIN вводится кнопками GUI. PIN не вводится в чат, не пишется в title, не логируется открыто.
- `atm_audit` переведён на `created_at`, чтобы не было ошибки `column "time" of relation "atm_audit" does not exist`.
- Live sidebar исправлен на нормальный русский текст без кракозябр и с попыткой скрыть технические score-цифры через Paper/Bukkit number format.

## Перед стартом

| № | Что проверить | Куда нажать / что выполнить | Что должно произойти | Фактический результат |
|---|---|---|---|---|
| 1 | `.env` на Ubuntu | Открыть `/opt/copimine/admin-web/.env` | Есть `POSTGRES_HOST`, `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_SCHEMA`; пароль не пустой и не `CHANGE_ME` | |
| 2 | systemd Minecraft | `systemctl cat copimine-minecraft.service` или свой unit Minecraft | В unit есть `EnvironmentFile=/opt/copimine/admin-web/.env` | |
| 3 | PostgreSQL | `systemctl status postgresql` | PostgreSQL запущен, порт `127.0.0.1:5432` открыт | |
| 4 | Миграции | Запустить миграции из `/opt/copimine/db/migrations` | Все миграции применились без ошибок, включая `20260611_003_bank_atm_audit_gui_fix.sql` | |
| 5 | Плагины на месте | Проверить `/opt/copimine/minecraft/server/plugins` | Есть `CopiMineUltimateAdminPlus.jar`, `CopiMineArtifacts.jar`, `CopiMineNarcotics.jar` | |

| № | Функция | Куда нажать / что выполнить | Что должно произойти | Фактический результат |
|---|---|---|---|---|

| 4 | Внести AR из руки | Взять официальный AR в руку → банкомат → `Внести предмет из руки` | AR исчезает из руки, баланс банка увеличивается, запись есть в ledger | |

| 6 | Снять 1 AR | Банкомат → `Снять 1 AR` → ввести PIN кнопками → `Подтвердить` | Баланс уменьшается на 1, в инвентарь выдан официальный AR | |
| 7 | Снять 16 AR | Банкомат → `Снять 16 AR` → PIN | Баланс уменьшается на 16, выдан официальный AR | |

| 9 | Неверный PIN | Ввести неправильный PIN | Операция отклонена русским сообщением, PIN не показан, попытка записана без PIN | |

| 11 | Очистить PIN | Нажать `Очистить` | Маска кода очищается | |
| 12 | Стереть цифру | Ввести несколько цифр → `Стереть цифру` | Удаляется последняя цифра маски | |

| 14 | Перевод самому себе | Попытаться выбрать себя | Система не даёт перевести самому себе | |
| 15 | Недостаточно AR | Попробовать снять/перевести больше баланса | Операция не проходит, баланс не уходит в минус | |
| 16 | Полный инвентарь при снятии | Заполнить инвентарь → снять AR | Система просит освободить слот, деньги не списываются | |
| 17 | Архив банкомата | В реестре банкоматов Shift+ПКМ по банкомату | Банкомат становится неактивным, ПКМ по блоку больше не открывает ATM | |

## CopiMineArtifacts

Проверяемые PostgreSQL-таблицы Artifacts: `artifact_items_catalog`, `artifact_shops`, `artifact_purchases`, `artifact_pending_deliveries`, `artifact_repairs`, `artifact_suspicious_events`.

| № | Функция | Куда нажать / что выполнить | Что должно произойти | Фактический результат |
|---|---|---|---|---|
| 1 | Команда обычного игрока | `/cmartifacts` без прав админа | Лавка не открывается; игрок видит подсказку, что лавка открывается кликом по блоку | |
| 1.1 | Админ-меню артефактов | `/cmartifacts admin` или `/cmartifacts` от OP/admin | Открывается служебное меню Artifacts: лавки, каталог, диагностика, reload | |
| 2 | Создать лавку | Смотреть на блок → `/cmartifacts shop create test_shop` | Создаётся лавка, запись появляется в `artifact_shops` | |
| 3 | Открыть лавку | ПКМ по блоку лавки | Открывается русская `Лавка артефактов` | |
| 4 | Категории | Нажать оружие/броня/инструменты/RP | Открывается категория с товарами и ценами; кнопки работают после перехода, session/shopId не теряются | |
| 4.1 | V4-каталог | Открыть оружие/броню/инструменты | Видны 12 товаров из V4: 4 оружия, 4 брони, 4 инструмента | |
| 4.2 | Скрытая вкладка рынка | Открыть лавку в обычный день | Кнопки `Чёрный рынок` нет, игрок не видит постоянный рынок | |

| 5 | Карточка товара | Нажать товар | Видно название, цену, редкость, перезарядку и оплату через банк | |

| 7 | PIN покупки | Если PIN нужен, ввести PIN кнопками | PIN маскируется, не пишется в чат/title/log | |
| 8 | Успешная покупка | Подтвердить при достаточном балансе | AR списаны через bridge, предмет выдан, ledger содержит операцию | |
| 9 | Недостаточно AR | Купить товар дороже баланса | Предмет не выдан, баланс не меняется | |
| 10 | Неверный PIN | Ввести неверный PIN | Покупка отклонена, попытка записана без PIN | |
| 11 | Double click | Быстро дважды нажать подтверждение | Нет двойного списания, idempotency сохраняет одну покупку | |
| 12 | Полный инвентарь | Заполнить инвентарь → купить | Создаётся pending delivery или покупка безопасно отклоняется по текущему сценарию, предмет не теряется | |
