# CopiMine Elections Manual Test Matrix

Заполняй колонку "Фактический результат" во время проверки на live-сервере.

| № | Раздел | Действие | Ожидаемый результат | Фактический результат |
|---:|---|---|---|---|
| 1 | Подготовка | Зайти админом, выполнить `/cmultra` | Открывается главное меню AdminPlus | |
| 2 | Подготовка | Открыть `Выборы` | Видны цикл, этапы, ЦИК, заявки, бюллетени, участки, аудит, sidebar | |
| 3 | Создание цикла | Нажать запуск/мастер жизненного цикла | Создаётся цикл, включается этап приёма заявок | |
| 4 | Приём заявок | Выдать книгу заявки игроку | Игрок получает официальную книгу заявки с защитной меткой | |
| 5 | Приём заявок | Игрок подписывает книгу | Заявка сохраняется, книга исчезает, игрок видит обычное русское сообщение | |
| 6 | Защита заявок | Игрок пытается подать вторую активную заявку | Вторая активная заявка не создаётся | |
| 7 | Ревью заявок | Админ открывает `Выборы` -> `Ревью заявок` | Видны поданные заявки, статусы и кнопки approve/reject | |
| 8 | Ревью заявок | Одобрить заявку | Статус становится APPROVED, действие пишется в audit | |
| 9 | Ревью заявок | Отклонить другую заявку | Статус становится REJECTED, кандидат не создаётся | |
| 10 | Кандидаты | Выполнить перенос approved -> candidates | Одобренный игрок появляется кандидатом один раз | |
| 11 | Кандидаты | Повторить перенос approved -> candidates | Дубликат кандидата не создаётся | |
| 12 | Участки | Создать участок по блоку | Блок регистрируется как активный участок ЦИК | |
| 13 | Участки | ПКМ по участку обычным игроком | Открывается пользовательское GUI участка | |
| 14 | Бюллетени | Открыть этап выдачи бюллетеней | Этап меняется на BALLOTS_OPEN | |
| 15 | Бюллетени | Выдать бюллетень игроку | Игрок получает личный официальный бюллетень | |
| 16 | Бюллетени | Попробовать использовать чужой/обычный лист бумаги | Голосование не открывается, игрок видит безопасное русское сообщение | |
| 17 | Голосование | Открыть этап голосования | Этап меняется на VOTING_OPEN, sidebar обновляется | |
| 18 | Голосование | Игрок выбирает кандидата в бюллетене | Бюллетень становится запечатанным, голос ещё не засчитан | |
| 19 | Голосование | Игрок опускает запечатанный бюллетень в участок | Голос записывается один раз, бюллетень удаляется, audit обновляется | |
| 20 | Защита голосования | Быстро кликнуть/повторно опустить бюллетень | Второй голос не появляется | |
| 21 | Защита голосования | Попробовать старый использованный бюллетень | Бюллетень отклоняется | |
| 22 | Подсчёт | Перевести выборы в подсчёт | Ledger синхронизируется с кандидатами | |
| 23 | Результаты | Завершить выборы | Победитель определяется по ledger и назначается президентом | |
| 24 | Президент | Президент открывает кабинет/мандат | Доступны президентские действия без доступа к сырым голосам | |
| 25 | Sidebar | Включить/перезагрузить live sidebar | Sidebar на русском, без красных score-цифр и кракозябр | |
| 26 | Права | Обычный игрок пытается управлять выборами | Управление запрещено | |
| 27 | Права | ЦИК/куратор открывает разрешённые разделы | Доступны только разрешённые действия | |
| 28 | Ошибки | Нажать действие не на том этапе | Игрок видит безопасное сообщение без SQL/Java/stacktrace | |
| 29 | PostgreSQL | Проверить таблицы и индексы выборов | Есть уникальные ограничения на голос, кандидата, активную заявку, бюллетень и участок | |
| 30 | Логи | После теста проверить latest.log | Нет PSQLException, SQLException, duplicate key, NullPointerException, ClassCastException | |
